## Dev Notes

### Lesson Structure Template
```yaml
---
id: lesson_a1_XX
title: "Lesson Title in Spanish"
level: A1
xp_value: 100
sequence_order: XX
grammar_points:
  - id: gp_identifier
    name: "Grammar Point Name"
    overview: "Brief explanation"
    examples:
      - "Example sentence 1"
      - "Example sentence 2"
vocabulary:
  - word: "spanish_word"
    translation: "english translation"
    example: "Example sentence using the word"
exercises:
  - type: fill_blank
    prompt: "Sentence with ___ blank"
    answer: "correct_word"
    xp_value: 10
  - type: multiple_choice
    prompt: "Question?"
    answer: "correct_option"
    options: ["option1", "option2", "option3", "option4"]
    xp_value: 10
---
# Lesson Title

## Introduction
Brief intro to lesson topic

## Grammar Point
Explanation of grammar concept

## Vocabulary
Additional context for key words

## Practice
Prompts for using new grammar/vocab
```

### Content Guidelines
- **Grammar Points:** Focus on one core concept per lesson (SER, possessives, numbers)
- **Vocabulary:** 5-8 high-frequency words related to lesson theme
- **Exercises:** Mix fill-in-blank (grammar practice) and multiple choice (vocab recognition)
- **Markdown:** Keep concise, use headers for structure, include tables for conjugations

### A1 Level Focus
Lessons target absolute beginners:
- Present tense verbs only
- Basic sentence structure (subject + verb + complement)
- High-frequency vocabulary (greetings, family, numbers)
- Simple, concrete concepts (no subjunctive, no complex tenses)

### Testing

**Test File Location:** No dedicated test file (content validation via parser)

**Validation Steps:**
1. **YAML Syntax:** Run parser, verify no parse errors
2. **Required Fields:** Parser validates all required fields present
3. **Database Insertion:** Query Supabase after seeding:
   ```sql
   SELECT * FROM lessons WHERE level = 'A1' ORDER BY sequence_order;
   SELECT * FROM grammar_points;
   SELECT * FROM exercises WHERE lesson_id IN (SELECT id FROM lessons WHERE level = 'A1');
   ```
4. **Content Quality:** Manual review for grammar accuracy and pedagogical soundness

**E2E Testing (after UI built):**
- Navigate to lesson in UI
- Verify Markdown renders correctly
- Verify vocabulary list displays
- Verify exercises are interactive and validate correctly
